# FACT v1.0.0（手动录入型 MVP）发布说明

发布日期：2026-04-25  
版本类型：MVP（手动录入型）

## 一、版本概述

v1.0.0 完成了 FACT 的“手动录入型 MVP”闭环：前端可新增舆情并触发分析，后端完成落库、风险预警生成与数据大屏汇总展示；模型服务以 mock 逻辑提供稳定的推理返回格式，便于后续接入真实模型。

## 二、技术栈

- **前端**：Vue 3 + Vite + TypeScript + Element Plus + Vue Router + Pinia + Axios + ECharts
- **后端**：Django + Django REST Framework + SQLite（开发）
- **模型服务（推理）**：FastAPI（mock 推理；不加载真实模型）
- **联调**：Vite Proxy + Django CORS 白名单（开发环境）

## 三、已完成模块与能力

### 1) fact_model_service（已完成）

- 接口：`/health`、`/predict/rumor`、`/predict/sentiment`、`/predict/full`
- 特点：mock 规则推理、固定 JSON 返回结构，供 Django 调用

### 2) fact_backend（已完成 MVP 闭环）

- 舆情数据：新增/列表/详情
- 分析闭环：`POST /api/opinions/{id}/analyze/`
  - 调用模型服务 `/predict/full`
  - 生成 `AnalysisResult` 与 `RiskWarning`
  - 幂等：重复分析不重复插入
- 关键词体系
  - `OpinionData.keywords`：入库关键词（人工/前端提取）
  - `AnalysisResult.keywords`：模型分析关键词
  - `display_keywords`：dashboard 统一展示关键词（合并两者，最多 8 个）
  - 管理命令：`python manage.py sync_opinion_keywords` 补齐历史关键词
- Dashboard：`GET /api/dashboard/summary/`（总数、最近舆情/预警等）

### 3) fact_frontend（已完成 MVP 展示）

- Dashboard：展示统计卡片、最近舆情/预警、基础图表
- 舆情页：
  - 表单校验 + 更完整字段
  - 关键词手动输入 + 一键自动提取
  - 支持自动分析
  - 状态中文标签化展示
- 分析结果页：中文化标签、概率/时间格式化、关键词标签展示
- 风险预警页：状态中文化、分数/时间格式化

## 四、当前限制（v1.0.0 范围外）

- 未实现登录注册与权限体系（仅保留最小 users 探活）
- 未接入真实深度学习模型（推理为 mock）
- 未接入真实爬虫采集与任务调度（仅保留 crawler_tasks 占位）
- 未做分页/复杂筛选/导入导出等完善的管理功能
- 仅为开发环境配置，不含生产部署方案

## 五、下一版本计划（v1.1.0 建议）

- **数据接入**：CSV 导入、爬虫数据经 Django API 入库的最小链路
- **分析增强**：模型服务接入真实模型（或可切换 mock/real）、支持模型版本记录与启用切换
- **预警增强**：风险评分策略可配置、增加预警原因模板与解释字段
- **前端增强**：筛选/分页/详情联动、预警处置（governance）最小流程页面
- **工程化**：统一配置（环境变量）、日志与错误追踪、接口类型自动生成（可选）
